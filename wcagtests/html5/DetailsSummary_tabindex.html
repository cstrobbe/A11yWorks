Findings (on Windows 7, 32-bit version):

  • The tabindex="0" makes the details element keyboard navigable in Internet Explorer 9.0.8112.16421, Firefox 12.0, Opera 11.64 (with the Tab key; on focus a blue outline is created around the summary element (just the width of the word) when details has no open attribute, and around the details element when when that element has an open attribute), SeaMonkey 2.1.0.1, Safari 5.1.7 (with the Tab key, and a blue outline is created around the summary element (full width of the paragraph) in the details element without the open attribute, and around the details element when the open attribute is present)
  • … but that does not make these browsers "support" details and summary: closing and expanding is still impossible; details always remains open.
